# Бот "Выгодные покупки"

Telegram-бот: пользователь присылает список продуктов, бот ищет их в Google Таблице
и отвечает, где дешевле, с учётом опечаток в названиях (fuzzy-поиск).

## Как это работает

1. Пользователь пишет боту список товаров (в столбик с номерами или через запятую).
2. Бот парсит список, сравнивает каждое название с товарами в таблице (нестрогое совпадение).
3. Для каждого товара выбирается магазин с минимальной ценой.
4. Если в таблице есть "Обычная цена" и текущая цена ниже — бот подсвечивает скидку.

## Структура Google Таблицы

Первая строка — заголовки (регистр и порядок важны):

| Товар | Магазин | Цена | Обычная цена | Ед. изм. | Обновлено |
|-------|---------|------|---------------|----------|-----------|
| Хлеб  | Lotus's | 35   | 42            | шт       | 2026-07-01|
| Хлеб  | Big C   | 38   | 38            |          |           |
| Молоко| Tesco   | 55   | 55            | л        |           |

Обязательны: **Товар, Магазин, Цена**. Остальное — опционально.
Один товар может встречаться в нескольких строках (по одной на магазин).

## Настройка (шаг за шагом)

### 1. Создать бота в Telegram
1. Напишите **@BotFather** в Telegram.
2. `/newbot` → укажите имя и username.
3. Скопируйте токен вида `123456789:AA...` — это `TELEGRAM_BOT_TOKEN`.

### 2. Дать боту доступ к Google Таблице
1. Зайдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте проект (или используйте существующий).
3. Включите **Google Sheets API** и **Google Drive API**
   (APIs & Services → Library → найти → Enable).
4. Создайте сервисный аккаунт: APIs & Services → Credentials → Create Credentials →
   Service Account.
5. Откройте созданный сервисный аккаунт → вкладка Keys → Add Key → JSON.
   Скачанный файл переименуйте в `credentials.json` и положите в папку проекта.
6. В самом JSON-файле найдите поле `client_email` (что-то вроде
   `имя@проект.iam.gserviceaccount.com`).
7. Откройте вашу Google Таблицу → кнопка "Настройки доступа" → добавьте этот email
   с правом **Читатель (Viewer)**.
8. Скопируйте ID таблицы из адресной строки:
   `https://docs.google.com/spreadsheets/d/ЭТОТ_КУСОК/edit` → это `GOOGLE_SHEET_ID`.

### 3. Настроить проект

```bash
python3 -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Откройте `.env` и впишите:
- `TELEGRAM_BOT_TOKEN` — токен от BotFather
- `GOOGLE_SHEET_ID` — ID таблицы
- `GOOGLE_SHEET_NAME` — название вкладки (листа) с данными
- `GOOGLE_CREDENTIALS_PATH` — путь к `credentials.json` (по умолчанию рядом с bot.py)

### 4. Запустить локально

```bash
python bot.py
```

Напишите боту `/start`, затем пришлите список продуктов.

## Деплой на Render

Важный момент: на бесплатном тарифе Render доступны только **Web Service**
(засыпает после 15 минут простоя, просыпается ~30-60 сек при новом запросе).
Тип сервиса **Background Worker** на бесплатном плане недоступен вообще.

Поэтому бот запускается в режиме **webhook**, а не polling: Telegram сам
"будит" сервис, присылая новое сообщение как HTTP-запрос. Код уже это
поддерживает — переключение happens автоматически, если задана переменная
`WEBHOOK_URL`.

### Шаг 1. Выложить код на GitHub

```bash
cd pricebot
git init
git add .
git commit -m "Первая версия бота"
```

Создайте пустой репозиторий на GitHub и запушьте:

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/pricebot.git
git branch -M main
git push -u origin main
```

⚠️ `.env` и `credentials.json` не должны попасть в репозиторий — секреты
задаются прямо в Render, а не хранятся в коде. Создайте файл `.gitignore`:

```
.env
credentials.json
venv/
__pycache__/
```

### Шаг 2. Создать Web Service на Render

1. Зайдите на [render.com](https://render.com) → New → Web Service.
2. Подключите GitHub-репозиторий с ботом.
3. Render автоматически подхватит `render.yaml` (Blueprint) — либо настройте вручную:
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Plan**: Free
4. В разделе **Environment** добавьте переменные:
   - `TELEGRAM_BOT_TOKEN` — токен от BotFather
   - `GOOGLE_SHEET_ID` — ID таблицы
   - `GOOGLE_SHEET_NAME` — название листа
   - `GOOGLE_CREDENTIALS_JSON` — вставьте **всё содержимое** `credentials.json`
     одной строкой (это надёжнее, чем загружать файл на Render)
   - `WEBHOOK_URL` — оставьте пустым на первый деплой

### Шаг 3. Прописать WEBHOOK_URL

После первого деплоя Render выдаст адрес вида
`https://pricebot-abcd.onrender.com`. Скопируйте его, добавьте в переменную
`WEBHOOK_URL` (без слэша на конце) и нажмите **Manual Deploy → Deploy latest commit**,
чтобы бот перезапустился с webhook-режимом.

### Шаг 4. Проверить

Напишите боту `/start` в Telegram. Если сервис "спал" — первый ответ может
прийти с задержкой до минуты, дальше отвечает быстро, пока не заснёт снова
от простоя.

Если хочется без задержек на пробуждение — платный план **Starter ($7/мес)**
убирает засыпание. Для личного использования бесплатного тарифа обычно достаточно.

## Возможные доработки

- Уведомления о снижении цены без запроса пользователя (по расписанию).
- Хранение списка покупок пользователя между сессиями (сейчас список не сохраняется).
- Учёт нескольких листов / нескольких таблиц (разные категории товаров).
- История цен для графика "цена со временем".
